# Terraform Lab 01: Local File Resource

## Goal

Learn Terraform fundamentals using a zero-cost local resource.

This lab uses the `local_file` resource to create a file on your machine.

---

## Concepts Covered

- Terraform provider
- Terraform resource
- `terraform init`
- `terraform plan`
- `terraform apply`
- Terraform state
- Drift detection
- `terraform destroy`

---

## Lab Structure

```text
labs/terraform/01-local-file/
├── main.tf
├── README.md
├── .terraform.lock.hcl
└── devops-note.txt
```

> `devops-note.txt` is generated by Terraform and should not be manually managed.

---

## Terraform Configuration

```hcl
terraform {
  required_version = ">= 1.5.0"

  required_providers {
    local = {
      source  = "hashicorp/local"
      version = "~> 2.5"
    }
  }
}

resource "local_file" "devops_note" {
  filename = "${path.module}/devops-note.txt"
  content  = "Terraform created this file for DevOps learning.\n"
}
```

---

## Commands Used

### Initialize Terraform

```bash
terraform init
```

Downloads the required provider plugin.

---

### Preview Changes

```bash
terraform plan
```

Shows what Terraform will create, update, or destroy.

---

### Apply Changes

```bash
terraform apply
```

Creates the resource defined in `main.tf`.

For non-interactive local testing:

```bash
terraform apply --auto-approve
```

---

### Check State

```bash
terraform state list
```

Expected output:

```text
local_file.devops_note
```

Show full state details:

```bash
terraform show
```

---

## Drift Test

Manual change outside Terraform:

```bash
echo "Manual change outside Terraform" > devops-note.txt
```

Run:

```bash
terraform plan
```

Terraform detects that the real file no longer matches the desired configuration.

Fix drift:

```bash
terraform apply
```

Terraform restores the file content from `main.tf`.

---

## Destroy Resource

```bash
terraform destroy
```

For non-interactive local testing:

```bash
terraform destroy --auto-approve
```

This removes the file managed by Terraform.

## Important Files

| File | Commit? | Reason |
|---|---:|---|
| `main.tf` | Yes | Terraform configuration |
| `.terraform.lock.hcl` | Yes | Locks provider versions |
| `.terraform/` | No | Downloaded provider plugins |
| `terraform.tfstate` | No | Local state file, may contain sensitive data |
| `devops-note.txt` | No | Generated resource |

---

## Key Learning

Terraform compares:

```text
Desired state in .tf files
vs
Current real infrastructure
vs
Terraform state
```

If something changes manually, Terraform can detect drift during `plan` and reconcile it during `apply`.

---

## Interview Summary

Terraform is an Infrastructure as Code tool. It uses configuration files to define the desired infrastructure state and a state file to track what it manages.

In this lab, I created a local file using Terraform, tested drift by manually changing the file, and used Terraform to restore the desired state.
